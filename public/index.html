Welcome to Toys API!

Read this documentation to use the API effectively.

GET an information about our toys:

To get all the toys use: /toys
To search a toy by name or info: /toys/search/?s=your-key-word
To get all the toys of the category: /toys/category/category-name
To get a range of prices: /toys/prices/?min=minimum-number&max=maximum-number
To get a specific toy by id: /toys/single/:id

You can add this to every GET request you make (of course, except searching by id):

Choose the page you would like to see (10 items per page): /toys/?page=number
Sort items: /toys/?sort=key-word
Reverse the items by sort key: /toys/?reverse=yes

SIGN UP and LOG IN to add, update or delete items

(A registered user can update or delete only the items which were added by this user)

Sign up:

Use this route to sign up: /users

You need to make a POST request giving the following information:
  • name (required, must include at least 2 characters, maximum 99 characters)
  • email (required, must include at least 6 characters, maximum 99 characters)
  • password (required, must include at least 6 characters, maximum 99 characters)
  • role (must include at least 4 characters, maximum 99 characters)

Log in:

Use this route to log in: /users/login

You need to make a POST request with the following information:
  • email (required, must include at least 6 characters, maximum 99 characters)
  • password (required, must include at least 6 characters, maximum 99 characters)
If you log in successfully, you will get token.

Add an item to the database by making a POST request:

(You need to log in, get a token and send it through headers to add an item)

You need to send the following information about an item to the router /toys:
  • name (required, must include at least 2 characters, maximum 99 characters)
  • info (required, must include at least 2 characters, maximum 999 characters)
  • category (required, must include at least 2 characters, maximum 20 characters)
  • img_url (must include at least 2 characters, maximum 999 characters)
  • price (required, must be at least 1, maximum 2000)

Edit an item in the database by making a PUT request:

(You need to log in, get a token and send it through headers to edit an item)

You need to send the following information about an item to the router /toys/:id, and replace the :id in the router by the _id of the item you want to edit.
  • name (required, must include at least 2 characters, maximum 99 characters)
  • info (required, must include at least 2 characters, maximum 999 characters)
  • category (required, must include at least 2 characters, maximum 20 characters)
  • img_url (must include at least 2 characters, maximum 999 characters)
  • price (required, must be at least 1, maximum 2000)

Delete an item in the database by making a DELETE request:

(You need to log in, get a token and send it through headers to delete an item)

You only need to replace an :id in the router with the _id of an item you want to delete: /toys/:id:

Use the router: /users/userInfo
